# From syntax to intent

For most of computing history, programming has been an act of translation: understand the problem in human terms, design a solution in abstract terms, then render it in syntax a machine can execute. That friction is now collapsing.

> "The most profound shift in software engineering isn't a new language, framework, or cloud service. It's the transition from writing code to expressing intent, and trusting intelligent systems to translate that intent into working software." — *Introduction*

As of early 2026, 85% of professional developers regularly use AI coding agents, 51% use them daily, and an estimated 41% of all new code is AI-generated. The shift didn't happen overnight — it moved through stages:

1. **Autocomplete** — token prediction in the editor.
2. **Inline suggestions** — completing entire functions.
3. **Chat interfaces** — natural-language feature requests with working implementations back.
4. **Autonomous agents** — clone repos, plan multi-file changes, run them in sandboxes, run tests, submit pull requests, with no human typing a line of code.

The implications for the software development life cycle (SDLC) are profound — every phase, from requirements to deployment to maintenance, is being reshaped. But the transformation isn't uniform. It spans a spectrum from casual **vibe coding** (prompt and accept) to disciplined **agentic engineering** (AI as implementation engine inside a system of constraints, tests, and feedback loops, humans retaining oversight of architecture and quality).

The distinction matters in the room: telling a CTO your team is vibe coding a payment system should raise alarm bells. Telling them you practice agentic engineering — AI implementing under human-designed constraints, correctness guaranteed by test coverage — is a different conversation entirely.
