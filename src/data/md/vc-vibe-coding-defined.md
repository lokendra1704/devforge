# What is vibe coding?

In February 2025, Andrej Karpathy described an approach where you "fully give in to the vibes, embrace exponentials, and forget that the code even exists." A developer describes what they want in natural language, accepts the AI's output, and when something breaks, copies the error back into the prompt and asks for a fix.

The term went viral because it named something real — but it was applied so broadly ("is a senior engineer using an AI assistant to implement a well-specified feature 'vibe coding'?") that it began to lose meaning. By early 2026, Karpathy himself acknowledged the original framing was too narrow and introduced **agentic engineering** to describe the disciplined end of the same spectrum.

The key differentiator between the two is not *whether* you use AI — it's **how much structure, verification, and human judgment surrounds the AI's output.**

> "The right position on this spectrum depends on the stakes. A weekend prototype can be pure vibe coding. A production API handling financial transactions demands agentic engineering. Most real work falls somewhere in between, and the skill is knowing where to draw the line for each task." — *Applied Tip*
