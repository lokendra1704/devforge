# Experience Replay: Breaking Correlation

The breakthrough insight: instead of updating the network on consecutive game samples, **store all experiences in a replay buffer and sample randomly from it during training.**

## The Problem Experience Replay Solves

When learning online from consecutive game frames:

1. **Strong correlation** — frame t and frame t+1 differ only slightly. Training on them sequentially violates the i.i.d. assumption deep learning relies on, increasing gradient variance.

2. **Feedback loops** — if the best action is "move left," the agent will move left repeatedly, so the training distribution becomes dominated by "left-movement" samples. If the best action switches to "move right," the distribution abruptly shifts. This can cause the network to oscillate or diverge catastrophically.

3. **Inefficiency** — each sample is used only once. In supervised learning, you'd iterate over the same labeled examples many times.

## Experience Replay: The Fix

Store the last N transitions (s, a, r, s') in a replay memory D. During training, **sample a random minibatch from D** rather than using consecutive samples.

This provides three benefits:

- **Breaks correlation** — by randomizing the order, we reduce the variance of our gradient updates
- **Smooths the learning distribution** — by sampling from a pool of past behaviors, the training distribution becomes an average over many previous policies, avoiding oscillations
- **Data reuse** — each transition can be used in multiple weight updates, improving sample efficiency

The trade-off: we're now learning off-policy (the samples were generated by an older policy), which is why we need Q-learning rather than on-policy algorithms like REINFORCE.

## The Deep Q-Learning Algorithm

```
Initialize replay memory D with capacity N
Initialize Q-network with random weights θ
for each episode:
  Initialize state sequence s_1
  for each timestep t:
    With probability ε, select random action a_t
    Otherwise, select a_t = argmax_a Q(s_t, a; θ)
    Execute a_t, observe reward r_t and next state s_{t+1}
    Store transition (s_t, a_t, r_t, s_{t+1}) in D
    
    Sample random minibatch from D
    Compute target: y = r if terminal, else r + γ·max_a Q(s', a; θ)
    Update θ by gradient descent on (y - Q(s, a; θ))²
```

The key design choice: keep the network weights θ fixed when computing the target y. This stabilizes training by preventing the target from chasing a moving network. In practice, implementations sometimes use a separate "target network" whose weights are frozen and periodically copied from the main network.

## Why This Works

By combining random sampling with minibatch training, we've transformed Q-learning from a sequential, correlated process into a noise-reduced, i.i.d. one that deep networks can handle. The replay buffer absorbs the non-stationarity: even as the policy improves and the environment distribution shifts, the buffer contains a mixture of old and new behaviors, smoothing out the effective distribution.
